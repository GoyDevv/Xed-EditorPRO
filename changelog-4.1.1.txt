================================================================================
HOW IT WORKS (1-paragraph recap; full detail in phases.md)
================================================================================
Provider-agnostic OpenAI-compatible client: GET /models (verify + list), POST /chat/completions with
a tools[] schema array. The model returns tool_calls; the APP executes them locally (permission
dialog here) and feeds results back as tool messages, looping until final text. Tools run client-side;
the key only lets the model request them. Kiro's ksk_ key has no public HTTP endpoint, so use the
Custom provider pointed at a Kiro→OpenAI gateway, or OpenAI/Gemini/OpenRouter directly.
