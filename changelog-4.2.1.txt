Xed-Editor PRO — Changelog
Version 4.2.1 (versionCode 128)

Smooth drawer expand + Git tab UI overhaul (stage 1: local/JGit).

>>> BUILD NOTE: verified by GitHub Actions CI (no local JDK 21 / SDK here).

================================================================================
FIXED
================================================================================
- Drawer expand/collapse no longer "snaps": the sheet width now glides via an animated spring
  (core/main/.../components/ResponsiveDrawer.kt, getDrawerWidth).

================================================================================
GIT TAB — overhaul (core/main/.../git/GitTab.kt, GitViewModel.kt)
================================================================================
- Tabbed layout with animated horizontal transitions: **Code · Diff · Actions**.
- Code tab: the change lists (Conflicts / Changes / Untracked) as before; tapping a file now opens it
  in the Diff tab (expanded) instead of a modal.
- Diff tab: **collapsible per-file cards** showing the unified diff with CLI-style coloring — green
  "+" adds and red "-" removals with tinted backgrounds — plus a **+N / -M line count** per file.
  Diffs load lazily and cache; the cache clears on every status sync.
- Commit box now has a **Commit title** AND a separate **Description** field (was a single field);
  the commit message is composed as "title\n\ndescription".
- Warnings: an explicit force-push warning in the push dialog; existing discard confirmation kept.
- Overflow menu (⋮): **Origin manager** and a **Git settings** shortcut (deep-links to Git settings).
- Origin manager dialog: view/set the `origin` remote URL (JGit) and open the repo on the web.
- Actions tab: opens your repo's Actions on GitHub in the browser + a Git-settings shortcut.

- app/build.gradle.kts: 4.2.0 -> 4.2.1 (versionCode 127 -> 128).
