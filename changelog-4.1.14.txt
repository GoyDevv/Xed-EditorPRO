Version 4.1.14 (versionCode 121)

AI Agent — automatic Kiro setup (install + sign-in) with live progress, browser/token login, and
copyable errors.

>>> BUILD NOTE: does NOT build locally (JDK 17 here; needs JDK 21 + Android SDK/NDK). Verified ONLY
>>> by GitHub Actions CI. Runtime behaviour (sandbox install + kiro-cli login) needs ON-DEVICE testing.

================================================================================
NEW in 4.1.14
================================================================================
- core/main/.../exec/ShellUtils.kt: runUbuntuStreaming() — runs a sandbox command and streams
  stdout+stderr line-by-line (with an optional timeout) for live progress views.
- core/main/.../ai/KiroSetup.kt (new): automatic Kiro setup engine.
    • runInstall(): installs kiro-cli in the Linux sandbox (curl|bash, with a wget / apt-get curl
      fallback), then **resolves the binary path** (fixes the "command not found" / PATH problem) and
      adds it to ~/.bashrc PATH for future terminals. The resolved path is saved (AiPrefs.kiroCliPath).
    • runLogin(): runs "kiro-cli login" (browser device flow), streaming output; captures the sign-in
      URL. On completion, re-discovers credentials via KiroAuth.
    • useToken(): token login — store a pasted refresh token instead.
    • Compose state: step statuses, rolling log, phase, error, loginUrl.
- core/main/.../ai/KiroSetupUi.kt (new): the setup dialog — a 2-step checklist, a progress bar, a
  small scrollable **live output** view, an "Open browser" / "Copy link" card when a sign-in URL
  appears, a copyable error + "Copy log", and a "Use token" path.
- core/main/.../ai/AiTab.kt: "Set up Kiro automatically" on the setup screen and an "Automatic setup
  (install + sign in)" button in the provider dialog open the new flow.
- core/main/.../ai/AiPrefs.kt: kiroCliPath.
- app/build.gradle.kts: 4.1.13 -> 4.1.14 (versionCode 120 -> 121).

================================================================================
Notes / known limits (please test on-device)
================================================================================
- The kiro-cli installer must support the sandbox CPU arch. On ARM devices, if cli.kiro.dev ships
  x86_64-only binaries, the installed CLI may not run — in that case use **token login** (paste a
  refresh token from a desktop Kiro login) which needs no local CLI.
- "kiro-cli login" is run non-interactively; if it needs a terminal prompt, the sign-in URL still
  appears in the log/card to open in a browser, and token login is always available. All output is
  copyable for troubleshooting.
