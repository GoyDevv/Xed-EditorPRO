================================================================================
NEW in 4.1.4
================================================================================
- core/main/src/main/java/com/rk/ai/AiAgentService.kt (new)
  Foreground service shown while the agent is working (mirrors com.rk.runner.RunService /
  DependencyInstallService pattern, which CI already builds). Notification "AI agent · Working …"
  with a **Stop** action; keeps the turn alive if the app is backgrounded. Channel "ai_agent_channel",
  id 7423. Includes AiAgentBus (a same-process callback holder): the notification's Stop action calls
  AiAgentBus.onStop.
- core/main/src/main/java/com/rk/ai/AiViewModel.kt (edited)
  • init registers AiAgentBus.onStop = { stop() }.
  • send() calls AiAgentService.start(app, "Working · <model>") before the turn and
    AiAgentService.stop(app) when it ends; stop() also stops the service.
- app/src/main/AndroidManifest.xml: declared com.rk.ai.AiAgentService (foregroundServiceType=specialUse).
  (REQUEST/FOREGROUND_SERVICE perms already present from earlier services.)
- app/build.gradle.kts: 4.1.3 -> 4.1.4 (versionCode 110 -> 111).
- phases.md: agent terminal/notification item updated to [~] partial with what remains.

How it behaves: send a message → an ongoing "AI agent" notification appears; tapping it opens the app;
tapping Stop cancels the turn (AiAgentBus → AiViewModel.stop()). The notification clears when the turn
finishes or is stopped. run_command still runs in the Linux sandbox (ShellUtils.runUbuntu) within the
turn.
