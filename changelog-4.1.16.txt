Xed-Editor PRO — Changelog
Version 4.1.16 (versionCode 123)

AI Agent — "Gemini (Google login)" provider: use consumer Gemini via Google account (no API key).

>>> BUILD NOTE: does NOT build locally (JDK 17; needs JDK 21 + SDK/NDK). Verified by GitHub Actions CI.
>>> EXPERIMENTAL: uses Google's undocumented internal Gemini web API via account cookies. It has no
>>> native tool-calling (a text protocol is used), Google changes it often, and it may be against
>>> Google's ToS (your account, your choice). The Gemini API-key provider stays as the reliable path.

================================================================================
NEW in 4.1.16
================================================================================
- core/main/.../ai/AiProviders.kt: new provider GEMINI_WEB ("Gemini (Google login)").
- core/main/.../ai/GoogleLoginDialog.kt (new): full-screen WebView Google sign-in that captures the
  Gemini web cookies (__Secure-1PSID / __Secure-1PSIDTS) via CookieManager once you reach
  gemini.google.com (auto-captures, or tap "Capture").
- core/main/.../ai/GeminiWebClient.kt (new): GeminiWebAuth (cookie storage) + a client that fetches
  the session token (SNlM0e/bl) from gemini.google.com and calls the internal StreamGenerate endpoint,
  parsing the batchexecute response. Because the web API has no function-calling, tools are exposed to
  the model as text (<tool_call>{...}</tool_call>) and parsed back into real tool calls, so it can
  still act as an agent (best-effort).
- core/main/.../ai/AiClient.kt: routes GEMINI_WEB to GeminiWebClient (chat + verify); cancel() aborts it.
- core/main/.../ai/AiViewModel.kt: isKeylessProvider() — GEMINI_WEB (like Kiro native) needs no key;
  isConfigured() true once cookies are captured.
- core/main/.../ai/AiPrefs.kt: geminiPsid / geminiPsidts.
- core/main/.../ai/AiTab.kt: "Sign in with Google (Gemini)" on the setup screen and in the provider
  dialog open the WebView login.
- app/build.gradle.kts: 4.1.15 -> 4.1.16 (versionCode 122 -> 123).

================================================================================
How to use
================================================================================
AI tab → "Sign in with Google (Gemini)" → sign in → it captures your login → pick "Gemini (Google
login)" as the provider and chat. No API key. If replies fail to parse (Google changed the internal
format) or sign-in expires, re-login — or switch to the Gemini API-key provider for a solid,
tool-native experience.

Needs ON-DEVICE testing (WebView cookie capture + the private web protocol can't be verified here).
