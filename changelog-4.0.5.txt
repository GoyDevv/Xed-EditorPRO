Xed-Editor PRO — Changelog
Version 4.0.5 (versionCode 104)

Android workflow + dependency manager overhaul.

================================================================================
0. Note on the recurring "utils: No such file or directory" log
================================================================================
That log is from a build BEFORE the 4.0.4 fix. The background run path (RunService) bypassed
MkSession, so the helper scripts the run script sources weren't installed. 4.0.4 added
`setupTerminalFiles()` to ProjectRunner.run()/sync(); this build keeps that fix. Once you build
4.0.4/4.0.5, Run/Sync no longer fail with "utils not found".

================================================================================
1. Minimal Android template (only MainActivity.kt)
================================================================================
- core/main/.../projects/ProjectScaffolder.kt — the Android (Compose) template now generates a
  single Kotlin file, MainActivity.kt, which is self-contained (uses MaterialTheme directly). The
  ui/theme/{Color,Theme,Type}.kt package and the ExampleUnitTest/ExampleInstrumentedTest files were
  removed, so a new empty project has exactly one .kt file like you expected.

================================================================================
2. Android Run/Sync behave like Android Studio
================================================================================
- core/main/.../runner/ProjectRunner.kt — Android projects must have a successful Gradle **Sync**
  once per session before Run will build. If you press Run first, you get a clear, detailed warning
  explaining to Sync first (and to install the SDK/NDK from Dependencies if Sync fails). A
  successful Sync (or build) marks the project synced for the session and unblocks Run.
- core/main/.../runner/RunService.kt — marks the project synced on a successful gradle run, and
  still installs the APK on a successful build.
- Gradle prints BUILD SUCCESSFUL / BUILD FAILED, which now shows clearly in the build output and
  the collapsed bar's status line.

================================================================================
3. Build output reads as proper log lines
================================================================================
- core/main/.../runner/RunOutputView.kt — the expanded output no longer soft-wraps; it scrolls
  horizontally so each log line stays on one line (no more words spread across lines). Compact
  monospace styling kept. The latest line in the collapsed bar slides up like a streaming log, and
  the expand/collapse uses a smooth spring.

================================================================================
4. Dependencies dialog overhaul (Android)
================================================================================
- core/main/.../projects/DependenciesDialog.kt —
  * Fuller Android SDK install: cmdline-tools + platform-tools + platforms android-35 & android-34
    + build-tools 35 & 34.
  * New **Android NDK** entry with a **version dropdown** (27/26/25/23 lines) — pick/switch the NDK
    version and it installs `ndk;<version>` via sdkmanager.
  * New **CMake** entry (cmake;3.22.1) for native builds.
  * Real-time, single-line output of the running install, an overall progress bar, and clear
    per-item Installed / Failed status.
- core/main/.../projects/DependencyInstallService.kt — installs now stream output line-by-line
  (via ubuntuProcess) into the dialog instead of buffering, and report overall progress. (No longer
  uses the buffered ShellUtils.runUbuntu for installs.)

================================================================================
5. Version
================================================================================
- app/build.gradle.kts — versionName 4.0.4 -> 4.0.5, versionCode 103 -> 104.

================================================================================
Android end-to-end flow now
================================================================================
1. Create/open an Android project.
2. Dependencies dialog → install Android SDK, NDK (pick version), CMake, a JDK.
3. Tap Sync (next to Run) → wait for BUILD SUCCESSFUL.
4. Tap Run → builds assembleDebug and installs the APK (system prompt).

Local build wasn't possible in the dev environment; verification is via GitHub Actions CI.
