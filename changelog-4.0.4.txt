Xed-Editor PRO — Changelog
Version 4.0.4 (versionCode 103)

Fixes the broken Run/Sync and polishes the build output.

================================================================================
1. FIX: Run/Sync failed with "utils: No such file or directory"
================================================================================
Root cause: the background run path added in 4.0.1 (RunService via ubuntuProcess) bypasses
MkSession, which is what normally installs the terminal helper scripts. So `project_runner`
sourced `$LOCAL/bin/utils` (which provides info/warn/error/command_exists) but that file was
never written — every helper call failed with "command not found", breaking all builds.

- core/main/.../runner/ProjectRunner.kt — `run()` and `sync()` now call `setupTerminalFiles()`
  (idempotent) before launching, so `utils` and the other helper scripts are present. Builds work
  again.

================================================================================
2. FIX: build output not shown when no file is open
================================================================================
- core/main/.../activities/main/MainContent.kt — the floating build view was nested inside the
  "files open" branch, so running from the no-file toolbar button produced no visible output. The
  content is now wrapped in a Box and the build view is an always-on overlay, shown whenever a
  build is active regardless of whether any file is open.

================================================================================
3. Removed the bottom extra-keys bar by default + added a toggle in the 3-dot menu
================================================================================
- core/main/.../settings/Settings.kt — `show_extra_keys` now defaults to OFF.
- core/main/.../commands/editor/ToggleExtraKeysCommand.kt (new) — an "Extra keys" toggle
  (ToggleableCommand) that turns the bottom <  ^  > bar on/off. Registered in CommandProvider,
  added to the default editor toolbar, and injected into existing users' toolbars via a one-time
  migration (migrated_extra_keys_toolbar). It shows highlighted when on, and sits in the editor
  toolbar's overflow (3-dot) menu.

================================================================================
4. Build output UI polish (minimised + maximised)
================================================================================
- core/main/.../runner/RunOutputView.kt —
  * Output now uses a compact monospace style (12sp, 16sp line height, 0 letter spacing) with
    tighter padding, so log lines read together as one block instead of being widely spaced.
  * The collapsed bar's latest line now slides up (stream-style) as new output arrives, instead of
    a plain cross-fade — reads "one line after another".
  * Expand/collapse spring retuned to dampingRatio 0.85 / StiffnessMedium: snappier and smoother,
    fixing the laggy minimise.

================================================================================
5. Version
================================================================================
- app/build.gradle.kts — versionName 4.0.3 -> 4.0.4, versionCode 102 -> 103.

================================================================================
Notes
================================================================================
- Existing users keep their toolbar; the migration just inserts the new "Extra keys" toggle. New
  installs get it (and the Android Sync button from 4.0.3) by default.
- Local build wasn't possible in the dev environment; verification is via GitHub Actions CI.
