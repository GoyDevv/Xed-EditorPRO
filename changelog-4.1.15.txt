================================================================================
FIXED / CHANGED in 4.1.15
================================================================================
- FIXED: the "login link" shown before install was actually the installer-script URL
  (cli.kiro.dev/install). Login URLs are now captured ONLY during the sign-in step and the installer
  URL is explicitly excluded.
- NEW coherent flow (core/main/.../ai/KiroSetup.kt, rewritten):
    1. CHECK — first detects whether kiro-cli already works ("kiro-cli/kiro/q --version") and whether
       you're already signed in (KiroAuth). If already connected, it just says so — nothing runs.
    2. INSTALL — only if the CLI isn't already present (shown as "already installed" / skipped).
    3. SIGN IN — only if not already signed in; browser device flow or token.
  autoConnect() runs just the steps that are actually needed.
- REWORKED dialog (core/main/.../ai/KiroSetupUi.kt): professional layout — header with icon, a plain
  explanation that Kiro is used as the agent DIRECTLY (no separate gateway), a 3-step checklist with
  status chips + one-line descriptions, a "Connected" banner when done, progress bar, a proper
  browser sign-in card (Open browser / Copy link) shown only for real login URLs, a token sign-in
  section, and a collapsible live-output log with Copy log. Auto-runs Check on open.
- core/main/.../exec/ShellUtils.kt: runUbuntuStreaming already supports a timeout (used for check 40s,
  install 900s, login 300s).
- app/build.gradle.kts: 4.1.14 -> 4.1.15 (versionCode 121 -> 122).

================================================================================
How Kiro works now (no gateway)
================================================================================
Once signed in, the app talks to Kiro/AWS CodeWhisperer DIRECTLY (KiroClient) — you do NOT need to run
any gateway. Just: AI tab → "Set up Kiro automatically" → Connect (or paste a token) → pick Kiro as the
model. On devices where the CLI binary can't run (CPU arch), token sign-in is the reliable path.
