NEW in 4.1.2 — streaming (feels like Claude Code)
================================================================================
- core/main/src/main/java/com/rk/ai/AiClient.kt (edited)
  Added chatStream(provider, key, model, messages, tools, onDelta): ChatResult.
  • Sends "stream": true + "stream_options": {"include_usage": true}.
  • Reads the SSE response line-by-line via resp.body.source().readUtf8Line(); parses "data: {…}"
    chunks, ignores "[DONE]".
  • Accumulates assistant text (calls onDelta(fragment) as each piece arrives) AND streamed
    tool-call fragments (id/name/arguments are accumulated per tool index, since arguments stream in
    pieces). Reads usage.total_tokens from the final chunk.
  • Returns the final assembled AiMessage (content + toolCalls) + total tokens.
  The older non-streaming chat() is kept (currently unused) as a fallback.

- core/main/src/main/java/com/rk/ai/AiViewModel.kt (edited)
  runAgentLoop now streams: it inserts a live placeholder assistant message, streams text into it via
  copy(content=…) (so the UI types it out + auto-scrolls), then replaces it with the final message
  (content + tool calls) before executing tools. Permission gate + persistence unchanged.

- app/build.gradle.kts: versionName 4.1.1 -> 4.1.2, versionCode 108 -> 109.
- phases.md: SSE streaming marked done; remaining Phase 2 items updated.
